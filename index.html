Flappy Mario

by: Nathan Vishnevetsky

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I wanted to make an expanded version of a game I already enjoyed. I thought it would be a good first foray into game development because the concept was inherently simplistic and could be innovated upon.

2. What I Learned

From user testing, I learned that there was a bug in my tutorial state for my game in which the pipes wouldn't spawn when loading from that state. I also learned that the backgrounds on my Mario and fireball objects were perceived by the testers to be very jarring, so I ended up removing them.

3. Architecture of My Game

The different classes in this game have many methods/functions that are made in one class and used in another by calling an instance of the class where the method was constructed. The architecture of the game contains about 8 different classes, one for each of the objects in the game, and a few classes which are used for creating the flow of the gameplay, interactions between objects, and updating the game's various components.

4. Technical Implementation

One part of the game I am especially proud of is the making of the fireballs which Mario has to dodge in order to survive. First, I created three different types of fireballs with different movement patterns, one of which spawns at random every few frames, to add variation and keep the player on their toes at all times. To calculate the angles for certain fireballs which home in on the player, I needed to use trigonometry(sin, cosine, tangent) and incorporate decimal values to make a precise angle for the fireballs to follow.

5. Most Challenging Part

The most challenging part of this project were the many issues and bugs that appeared in many points throughout the production process. For example, the addition of the fireballs into my game caused instances where the fireballs were impossible to dodge due to there being a pipe above or below them, so I needed to add an entirely new dash mechanic in order to nullify this issue. Also, when I was about to publish the game to create this website, I went back into the code and discovered that there was an issue in the user interface for the menus that I didn't notice earlier, so I had to go and delete my original downloaded JAR file for the site and upload an entirely new one to git, which was a big waste of time.

6. Future Fixes and Updates

In the future, I would like to make more smooth difficulty progression in my game by only spawning harder types of fireballs when the player reaches a certain score threshold. Also, I might update the game by creating a level-based progression system, which would introduce variations to the arc in which the pipes move and potentially even include bosses that the player can fight at designated points throughout the game.
